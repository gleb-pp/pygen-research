# Генераторы в Python: исследование структуры и границ применимости

В данном исследовании я погружаюсь в вопрос сериализации генераторов в Python. Я изучаю структуру PyGenObject, как генераторы хранят свой фрейм, как `yield` превращается в `YIELD_VALUE` в байткоде, и эволюцию от простых генераторов к `yield from`. Я также провожу бенчмарки по памяти и скорости переключения, исследую границы применимости генераторов и рассматриваю потенциальную сериализацию.

## Предисловие

Во время своей стажировки в МТС я столкнулся с задачей, лягшей в основую моего исследования.

У вас есть сервер, который обрабатывает сложные алгоритмы из нескольких задач, и хосты, которые могут выполнять простые задачи по запросу сервера. Сервер определяет, какую задачу необходимо выполнить, выслыает хосту запрос на её выполнение, хост выполняет её и возвращает результат серверу. 

При получении результата сервер должен продолжить выполнение алгоритма с того же места, на котором он остановился. Подобная задача идеальна для генератора в Python, который может приостанавливать выполнение функции, ожидать результат, а затем продолжать выполнение с того же места.

Однако одно из условий не позволяет нам использовать генераторы напрямую: мы не можем гарантировать, что сервер будет ожидать результат от хоста, находясь всё время рабочем состоянии, потому после отправки запроса необходимо сериализовывать состояние алгоритма, помещать его в базу данных, а при получении результата от хоста восстанавливать его и продолжать выполнение.

Здесь и возникает проблема: при использовании модуля `dill`, повсеместно используемого для сериализации объектов в Python, при попытке сериализации генератора возникает ошибка `TypeError: generator object is not serializable`. Вы можете изучить простой пример в файле `src/use_case.py`, который демонстрирует эту проблему.

```bash
git clone git@github.com:gleb-pp/pygen-research.git
cd pygen-research
pip install poetry
poetry install
poetry run python src/use_case.py
```

После изучения нескольких статей и обсуждения с нейросетями, я получил ответ о том, что генераторы в Python не поддерживают сериализацию, и что это ограничение не конкретного модуля `dill`, а языка. Тогда я подумал, что это довольно смелое заявление, ведь генераторы - это всего лишь объекты, и если `dill` может сериализовать функции, классы и даже замыкания, то в чем особенность генераторов? Это заставило меня глубже погрузиться в этот вопрос.
